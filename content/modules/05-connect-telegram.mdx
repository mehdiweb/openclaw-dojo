# Connect Telegram

Telegram is often easier than WhatsApp because it uses bot tokens instead of phone numbers.

**Time needed:** 5 minutes

## Step 1: Create a Bot with BotFather

1. Open Telegram on your phone or desktop
2. Search for: **@BotFather**
3. Start a chat
4. Send: `/newbot`
5. Follow prompts:
   - Name your bot (e.g., "My AI Assistant")
   - Choose username (must end in "bot", e.g., "myai_bot")
6. **Copy the token** BotFather gives you (looks like: `123456789:ABCdefGHIjklMNOpqrSTUvwxyz`)

**Save this token!** If you lose it, you'll need to create a new bot.

## Step 2: Configure OpenClaw

Edit your config:

```bash
nano ~/.openclaw/openclaw.json
```

Add Telegram section:

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "123456789:ABCdefGHIjklMNOpqrSTUvwxyz",
      "dmPolicy": "pairing"
    }
  }
}
```

**Replace** the token with yours.

Save: Ctrl+X, Y, Enter

## Step 3: Restart Gateway

```bash
openclaw gateway restart
```

Or if running manually, stop it (Ctrl+C) and start again:
```bash
openclaw gateway
```

## Step 4: Test It

1. Find your bot on Telegram (search for the username you created)
2. Send: `/start` or "Hello"
3. You should get a pairing code back

## Step 5: Approve Pairing

Check the code:
```bash
openclaw pairing list telegram
```

Approve it:
```bash
openclaw pairing approve telegram ABC123
```

## Step 6: Test Again

Send another message. You should get an AI response now!

## Optional: Allow Specific Users

Instead of "pairing" for everyone, you can allow specific users:

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "your-token",
      "dmPolicy": "allowlist",
      "allowFrom": ["123456789"]
    }
  }
}
```

**Get your user ID:**
- DM @userinfobot
- Or check logs after first message: `openclaw logs --follow`

## Group Setup (Optional)

To add your bot to a group:

1. Add bot to group
2. Make it admin (optional but recommended)
3. Configure group settings:

```json
{
  "channels": {
    "telegram": {
      "groups": {
        "-1001234567890": {
          "requireMention": true
        }
      }
    }
  }
}
```

**Get group ID:** Forward any message from the group to @userinfobot

## Troubleshooting

### "Invalid token" error
- Check you copied the full token
- Include the numbers before the colon
- No extra spaces

### Bot doesn't respond
- Check `openclaw logs --follow`
- Make sure gateway is running: `openclaw gateway status`
- Verify token is in config (not just env variable)

### "BotFather says token revoked"
- Someone used your token
- Create new bot with /newbot
- Update config with new token
